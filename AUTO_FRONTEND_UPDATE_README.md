# 自动化前端更新部署脚本

## 🎯 功能说明
这是一个智能化的前端更新部署工具，能够自动检测前端代码变更并触发相应的更新部署流程。

## 🚀 使用方法

### 自动模式（推荐）
```bash
node scripts/auto-frontend-update.js
```

脚本会自动执行以下操作：
1. 检查本地未提交的前端更改
2. 自动提交并推送更改
3. 检查服务器前端状态
4. 如需要则触发服务器更新
5. 验证更新结果

### 手动触发服务器更新
如果你只需要更新服务器端的前端而不涉及本地代码变更：
```bash
# 在服务器上手动重建前端容器
ssh -i "密钥路径" root@服务器IP "cd /root/smart-kitchen && docker compose build frontend && docker compose up -d frontend"
```

## 🔧 工作原理

### 检测机制
- **时间检测**: 通过比较前端构建文件的时间戳判断是否需要更新
- **内容检测**: 检查本地是否有未提交的前端代码变更
- **状态检测**: 验证服务器容器运行状态和页面可访问性

### 更新策略
1. **优先使用GitHub Actions**: 如果有代码变更，优先通过CI/CD流程更新
2. **备用手工更新**: 当自动流程失败时，提供手动更新选项
3. **多重验证**: 更新后进行多维度验证确保部署成功

## 📋 验证清单

脚本会自动验证以下项目：
- [x] 容器运行状态
- [x] 页面HTTP响应状态
- [x] 前端构建时间戳
- [x] 关键资源文件存在性

## ⚠️ 注意事项

### 使用前提
- 确保SSH密钥配置正确
- 服务器Docker环境正常运行
- 有足够的磁盘空间进行容器重建

### 故障处理
如果自动更新失败：
1. 检查网络连接和SSH配置
2. 手动执行 `docker compose build frontend`
3. 查看容器日志 `docker compose logs frontend`
4. 验证Nginx配置是否正确

## 🛠️ 配置说明

### SSH密钥配置
确保在脚本中正确配置SSH密钥路径：
```javascript
// 修改脚本中的SSH配置
'ssh -i "你的密钥路径" root@服务器IP ...'
```

### 服务器路径配置
确保服务器项目路径正确：
```javascript
// 默认路径
/root/smart-kitchen
```

## 📊 监控指标

脚本会输出以下关键信息：
- 本地代码状态
- 服务器构建时间
- 容器运行状态
- 页面访问验证结果

## 🔄 最佳实践

1. **定期执行**: 建议每天执行一次自动更新检查
2. **变更即更新**: 前端代码有重要变更时立即执行
3. **监控日志**: 保留执行日志用于问题排查
4. **备份策略**: 重要更新前做好数据备份
