# 厨房任务调度系统数据库初始化说明

## 📋 概述

本文档详细说明如何根据MVP文档和菜品库文档手动创建和初始化数据库结构。

## 🏗️ 数据库结构

### 核心表结构

1. **users** - 用户表
2. **stations** - 工位表  
3. **dish_categories** - 菜品分类表
4. **recipes** - 菜谱表
5. **dishes** - 菜品表
6. **orders** - 订单表
7. **order_items** - 订单菜品表

### 表关系设计

```
stations (1) ←→ (N) dishes (N) ←→ (1) dish_categories
dishes (1) ←→ (N) order_items (N) ←→ (1) orders
recipes (1) ←→ (0..1) dishes
```

## 🚀 初始化方式

### 方式一：使用Prisma种子脚本（推荐本地开发）

```bash
# 1. 确保在backend目录
cd backend

# 2. 运行种子脚本
npm run db:seed

# 或者直接运行
npx ts-node prisma/seed.ts
```

### 方式二：使用SQL脚本（推荐生产环境）

```bash
# 1. 使用Windows批处理脚本（自动部署到远程服务器）
deploy-database.bat

# 2. 或者手动执行SQL
# 在PostgreSQL客户端中执行 backend/prisma/init-database.sql
```

### 方式三：手动创建表结构

直接在数据库中执行 `backend/prisma/init-database.sql` 文件中的SQL语句。

### 数据库用户权限

确保数据库用户具有以下权限：
- CREATE TABLE
- INSERT
- SELECT
- UPDATE
- DELETE

## 🧪 验证初始化结果

执行以下SQL验证数据是否正确初始化：

```sql
-- 验证表数量
SELECT COUNT(*) as table_count FROM information_schema.tables 
WHERE table_schema = 'public';

-- 验证基础数据
SELECT '工位数量: ' || COUNT(*) FROM stations;
SELECT '菜品分类数量: ' || COUNT(*) FROM dish_categories;
SELECT '菜谱数量: ' || COUNT(*) FROM recipes;
SELECT '菜品数量: ' || COUNT(*) FROM dishes;
SELECT '订单数量: ' || COUNT(*) FROM orders;
```

## ⚠️ 注意事项

1. **备份数据**：在生产环境执行前务必备份现有数据
2. **权限检查**：确保数据库用户有足够的操作权限
3. **环境验证**：先在测试环境验证脚本正确性
4. **索引优化**：已为常用查询字段创建索引
5. **数据完整性**：使用外键约束保证数据一致性

## 📞 故障排除

### 常见问题

1. **权限不足**：检查数据库用户权限设置
2. **连接失败**：验证数据库连接参数
3. **重复数据**：使用 `skipDuplicates: true` 避免重复插入
4. **外键约束**：确保关联数据已存在
