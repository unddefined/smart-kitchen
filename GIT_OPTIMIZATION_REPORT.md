## 🔧 技术细节

### Git配置优化
```bash
# 本地仓库
git config core.autocrlf true  # Windows环境自动转换行结束符
git config push.default simple # 简化推送行为

# 服务器仓库
git config core.autocrlf input # Unix环境保持LF
git config receive.denyCurrentBranch updateInstead # 允许推送更新
```

### 文件结构优化
```
项目根目录/
├── .gitignore          # 完善的忽略规则
├── database/           # 数据库相关文档和脚本
│   ├── *.md           # 技术文档
│   └── *.sql          # SQL脚本
├── deploy/            # 部署脚本
├── scripts/           # 工具脚本

```

## 🎯 最佳实践建议

### 1. 日常开发流程
```bash
# 开始工作前
git pull origin main

# 开发过程中
git add .
git commit -m "feat/fix/chore: 具体变更描述"

# 结束工作前
git push origin main
```

### 2. 分支管理策略
- `main`: 生产环境稳定分支
- `develop`: 开发集成分支  
- `feature/*`: 功能开发分支
- `hotfix/*`: 紧急修复分支

### 3. 提交信息规范
```
类型(模块): 简洁描述

详细说明（可选）

关联issue: #123
```
