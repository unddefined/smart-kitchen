# 数据库更新日志

## 2024-03-01 - 添加菜品预处理需求字段

### 变更内容
- 在 `dishes` 表中添加 `need_prep` 字段（BOOLEAN，默认值 FALSE）
- 该字段用于标识菜品是否需要预处理（如裹粉、蒸、预炸等）
- 如果为 FALSE，则在订单处理流程中跳过 `status.prep` 状态

### 技术实现
1. **数据库层面**：
   - 执行 SQL 语句：`ALTER TABLE dishes ADD COLUMN need_prep BOOLEAN DEFAULT FALSE`
   - 添加字段注释说明用途
   - 验证字段添加成功

2. **应用层面**：
   - 更新 Prisma schema 添加 `needPrep` 字段定义
   - 重新生成 Prisma Client
   - 更新 DishesService 添加相关查询和更新方法
   - 更新 DishesController 添加新的 API 端点

3. **数据初始化**：
   - 设置部分菜品的预处理需求为 TRUE：
     - 托炉饼（需要预炸）
     - 红烧肉（需要预处理）
     - 椒盐排骨（需要裹粉预炸）
     - 小笼馒头（需要蒸制）

### 新增 API 端点
- `GET /dishes/prep/:needPrep` - 根据预处理需求筛选菜品
- `POST /dishes/batch-prep` - 批量更新菜品的预处理需求

### 验证结果
- ✅ 字段成功添加到数据库
- ✅ Prisma Client 重新生成成功
- ✅ 相关服务方法实现完成
- ✅ API 端点测试通过

### 影响范围
- **正面影响**：提升了订单处理流程的精确性，避免不必要的预处理步骤
- **注意事项**：现有订单处理逻辑需要相应调整以利用此字段
