# 单页面应用(SPA)优化说明

## 🔄 优化背景

根据项目实际需求，智能厨房管理系统应作为**单页面应用**运行，直接进入核心功能界面，无需首页导航。

## 🎯 优化内容

### 1. 路由结构调整
**变更前**:
```
/ -> HomeView (首页导航)
/overview -> OverviewView (厨房总览)
/order -> OrderView (订单管理)
/recipe/:id -> RecipeView (菜谱详情)
```

**变更后**:
```
/ -> OverviewView (厨房总览) ⭐ 默认首页
/order -> OrderView (订单管理)
/recipe/:id -> RecipeView (菜谱详情)
```

### 2. 文件清理
- ✅ 删除 `HomeView.vue` 组件文件
- ✅ 移除路由配置中的Home相关条目
- ✅ 清理所有对HomeView的引用

### 3. 用户体验提升
- **直接访问**: 用户打开应用即进入核心工作界面
- **操作效率**: 减少不必要的页面跳转
- **专注工作**: 直接进入厨房管理状态

## 🚀 技术实现

### 路由配置优化
```javascript
// frontend/src/router/index.js
const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    {
      path: '/',           // 根路径
      name: 'overview',    // 直接映射到厨房总览
      component: OverviewView
    },
    // ... 其他路由保持不变
  ]
})
```

### 应用结构简化
```
src/
├── App.vue              # 主应用组件
├── main.js             # 应用入口
├── router/
│   └── index.js        # 简化后的路由配置
├── views/
│   ├── OverviewView.vue # 默认首页 ✓
│   ├── OrderView.vue    # 订单管理
│   └── RecipeView.vue   # 菜谱详情
└── components/
    └── OrderInputModal.vue # 订单录入弹窗
```

**优化后**:

- **菜谱查看**: 从订单详情或搜索直接进入

## 🔧 开发者注意事项

### 路由守卫
如果需要添加认证或权限检查，应在OverviewView级别实现：

```javascript
// 在OverviewView.vue中
onMounted(() => {
  // 检查用户权限
  if (!hasPermission()) {
    router.push('/login')
  }
})
```

### 状态管理
- 应用状态通过Pinia在全局管理
- 页面间数据共享更加直接
- 避免了不必要的组件嵌套
