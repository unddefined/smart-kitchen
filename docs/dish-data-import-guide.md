# 智能厨房系统菜品数据录入说明

## 📋 概述

本文档说明如何将菜品库数据录入到云数据库中，以及相关的管理脚本使用方法。

## 🎯 目标

- 将 `菜品库.md` 中的菜品数据录入到云数据库
- 建立完整的菜品分类体系
- 正确标记需要按人数计数的菜品
- 确保数据完整性和一致性

## 🏗️ 数据库结构

### 主要表结构

1. **stations** (工位表) - 6个工位
   - 热菜、打荷、凉菜、蒸煮、点心、切配

2. **dish_categories** (菜品分类表) - 7个分类
   - 凉菜、前菜、中菜、点心、蒸菜、后菜、尾菜

3. **dishes** (菜品表) - 61道菜品
   - 包含菜品名称、所属工位、分类、是否计数等信息

## 🚀 使用方法

### 1. 自动化管理脚本

运行交互式管理脚本：
```cmd
scripts\cloud-dish-manager.bat
```

脚本提供以下功能：
- 录入菜品数据到云数据库
- 验证数据完整性
- 处理历史遗留数据
- 生成总结报告
- 执行完整管理流程

### 2. 单独执行脚本

#### 录入菜品数据
```cmd
cd backend
node setup-cloud-dish-data.js
```

#### 验证数据完整性
```cmd
cd backend
node verify-cloud-dish-data.js
```

#### 处理遗留数据
```cmd
cd backend
node handle-legacy-dishes.js
```

#### 生成总结报告
```cmd
cd backend
node generate-summary-report.js
```
