## 数据来源

菜品数据来自 `docs/菜品库.md`,包含以下分类:

- **凉菜**: 2 个菜品
- **前菜**: 8 个菜品
- **中菜**: 24 个菜品 (部分需要预处理或计数)
- **点心**: 2 个菜品 (计数)
- **蒸菜**: 8 个菜品 (部分计数)
- **后菜**: 5 个菜品 (部分需要预处理)
- **尾菜**: 2 个菜品

**总计**: 51 个菜品

### 特殊字段说明

#### countable(是否计数)
标记为"（计数）"的菜品，例如:
- 椒盐排骨（计数）
- 托炉饼（计数）
- 小笼馒头（计数）
- 蒜蓉小鲍鱼（计数）

这类菜品需要按用餐人数计算份量。

#### need_prep(是否需要预处理)
标记为"（需要预处理）"的菜品，例如:
- 红烧肉（需要预处理）
- 椒盐猪手（需要预处理）
- 松鼠桂鱼（需要预处理）

这类菜品需要先进行预处理 (如裹粉、蒸、预炸等)。

## 文件说明

### 1. SQL 脚本
**位置**: `database/insert-dishes.sql`

传统的 SQL 插入脚本，适合直接在数据库层面执行。

**特点**:
- 使用 `ON CONFLICT (name) DO NOTHING` 避免重复插入
- 自动关联工位和分类
- 自动设置 countable 和 need_prep 字段
- 包含数据验证查询

### 2. Prisma TypeScript 脚本 ⭐推荐
**位置**: `backend/src/prisma/seed-dishes.ts`

使用 Prisma ORM 的类型安全种子脚本，**推荐使用**。

**依赖**: 
- `@prisma/client`
- `@prisma/adapter-pg`
- `dotenv`

**优点**:
- ✅ 类型安全，编译时检查
- ✅ 使用 Prisma 官方推荐的适配器模式
- ✅ 幂等性设计，不会重复插入
- ✅ 详细的日志输出
- ✅ 符合项目技术规范

### 3. Node.js 脚本
**位置**: `backend/src/seed-dishes.js`

通过原生 pg 客户端执行 SQL 脚本。

### 4. Shell 脚本
**位置**: `scripts/init-dishes-data.sh` (Linux/Mac)
**位置**: `scripts/init-dishes-data.ps1` (Windows)

在服务器上直接执行 SQL 文件的脚本。

## 使用方法

### 方法一：使用 Prisma TypeScript 脚本（推荐）⭐

1. **配置环境变量**

在 `backend` 目录创建或编辑 `.env` 文件:
```bash
DATABASE_URL=postgresql://smart_kitchen_user:your_password@localhost:5432/smart_kitchen_prod
```

或者使用生产环境配置:
```bash
cp .env.production .env
# 然后编辑 .env 填入正确的数据库密码
```

2. **安装依赖**（如果还没安装）
```bash
cd backend
npm install
```

3. **生成 Prisma Client**（如果还没生成）
```bash
npx prisma generate
```

4. **运行种子脚本**
```bash
npm run seed:dishes
```

或者直接使用 npx:
```bash
npx ts-node src/prisma/seed-dishes.ts
```

### 方法二：使用 npm 脚本（根目录）

在项目根目录:
```bash
npm run seed:dishes
```

这会调用 `backend/src/seed-dishes.js`。

### 方法三：直接执行 SQL（高级用户）

1. **连接到数据库**

Docker 环境:
```bash
docker exec -it smart-kitchen-postgres psql -U smart_kitchen_user -d smart_kitchen_prod
```

本地环境:
```bash
psql -h localhost -U smart_kitchen_user -d smart_kitchen_prod
```

2. **执行 SQL 文件**

从命令行:
```bash
psql -h localhost -U smart_kitchen_user -d smart_kitchen_prod -f database/insert-dishes.sql
```

或在 psql 交互界面:
```sql
\i /path/to/database/insert-dishes.sql
```

### 方法四：使用服务器脚本（生产环境）

在服务器上:

1. **添加执行权限**（仅首次）
```bash
chmod +x scripts/init-dishes-data.sh
```

2. **运行脚本**
```bash
./scripts/init-dishes-data.sh
```

或使用 PowerShell（Windows）:
```powershell
.\scripts\init-dishes-data.ps1
```

### 数据库查询验证

连接数据库后执行:

```sql
-- 查看分类统计
SELECT 
    dc.name as 分类，
    COUNT(d.id) as 菜品数量，
    COUNT(CASE WHEN d.countable = TRUE THEN 1 END) as 计数菜品数，
    COUNT(CASE WHEN d.need_prep = TRUE THEN 1 END) as 需要预处理菜品数
FROM dish_categories dc
LEFT JOIN dishes d ON dc.id = d.category_id
GROUP BY dc.id, dc.name
ORDER BY dc.display_order;

-- 查看所有菜品
SELECT d.name as 菜品名称，dc.name as 分类，s.name as 工位，d.countable, d.need_prep
FROM dishes d
JOIN dish_categories dc ON d.category_id = dc.id
JOIN stations s ON d.station_id = s.id
ORDER BY dc.display_order, d.name;
```

## 故障排查

### 问题 1: DATABASE_URL 未配置

**错误信息**:
```
Error: Cannot find env variable DATABASE_URL
```

**解决方案**:
1. 检查 `backend/.env` 文件是否存在
2. 确认 `DATABASE_URL` 格式正确:
   ```
   DATABASE_URL=postgresql://用户名：密码@主机：端口/数据库名
   ```
3. 如果使用生产配置，复制 `.env.production` 并修改

### 问题 2: Prisma Client 未生成

**错误信息**:
```
@prisma/client did not initialize yet. Please run "npx prisma generate"
```

**解决方案**:
```bash
cd backend
npx prisma generate
```

### 问题 3: 数据库连接失败

**错误信息**:
```
Can't reach database server at `localhost:5432`
```

**解决方案**:
1. 确认 PostgreSQL 服务正在运行
   ```bash
   # Docker 环境
   docker ps | grep postgres
   
   # 本地环境
   systemctl status postgresql
   ```
2. 检查数据库凭据是否正确
3. 确认防火墙允许连接

### 问题 4: 唯一约束冲突

**错误信息**:
```
Unique constraint failed on the fields: (`name`)
```

**解决方案**:
这是正常现象，说明菜品已存在。脚本会自动跳过已存在的菜品，继续插入其他菜品。

### 问题 5: 工位或分类不存在

**错误信息**:
```
Foreign key constraint failed on the field: `station_id`
```

**解决方案**:
脚本会自动创建所需的工位和分类。如果仍然报错，请检查:
1. 数据库表结构是否正确
2. 是否有足够的权限插入数据

### 问题 6: TypeScript 编译错误

**错误信息**:
```
Cannot find module '@prisma/client'
```

**解决方案**:
```bash
cd backend
npm install
npx prisma generate
```

## 安全提示

⚠️ **重要**:
- ✅ 脚本采用增量式设计，不会删除现有数据
- ✅ 使用 upsert 操作确保幂等性
- ✅ 符合生产环境数据安全规范
- ❌ 不要在脚本中添加 deleteMany 等破坏性操作
- ❌ 不要手动修改已存在的菜品数据
