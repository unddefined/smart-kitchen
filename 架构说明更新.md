# 智能厨房管理系统架构说明（更新版）

## 🏗️ 最新架构设计

### 正确的组件层次结构

```
App.vue (根组件 - 包含全局NavBar)
├── Global Bottom Navigation (烹调/库存/待办/历史)
└── Router View (动态内容区域)
    ├── Cooking.vue (烹调模块)
    │   ├── Header区域
    │   │   ├── 用户信息
    │   │   ├── 日期餐次切换
    │   │   └── 功能按钮组
    │   ├── Body区域
    │   │   ├── Sidebar (左侧菜单)
    │   │   │   ├── 总览选项卡
    │   │   │   └── 订单选项卡
    │   │   └── Content Area (右侧内容)
    │   │       ├── OverviewView.vue (总览视图)
    │   │       └── OrderView.vue (订单详情视图)
    │   └── OrderInputModal (订单录入弹窗)
    ├── Store.vue (库存模块)
    ├── Todo.vue (待办模块)
    └── History.vue (历史模块)
```

## 🔧 关键设计决策

### 1. NavBar位置修正 ✅
- **位置**: App.vue 根组件下
- **作用**: 全局功能模块切换器
- **行为**: 点击切换路由，保持状态

### 2. 组件职责明确化 ✅
- **App.vue**: 全局布局 + 导航管理
- **Cooking.vue**: 烹调业务逻辑 + 局部UI
- **功能组件**: 各自独立的业务模块

### 3. 路由策略优化 ✅
- 每个功能模块对应独立路由
- 保持浏览器历史记录
- 支持直接URL访问

## 🎯 功能模块详解

### 1. 烹调模块 (/)
**核心特色**: 
- 实时订单管理
- 菜品状态跟踪
- 优先级调度系统
- 工位任务分配

**交互流程**:
1. 用户通过左侧菜单切换总览/具体订单
2. 右侧展示对应内容视图
3. 底部全局导航可切换到其他模块

### 2. 库存模块 (/store)
**功能定位**: 食材和菜品库存管理
- 菜品库存实时监控
- 食材采购计划管理
- 库存预警和补货提醒

### 3. 待办模块 (/todo)
**功能定位**: 任务和提醒中心
- 切配任务跟踪管理
- 特殊要求提醒系统
- 工作进度可视化

### 4. 历史模块 (/history)
**功能定位**: 数据统计和分析
- 订单历史查询
- 经营数据报表
- 性能分析仪表板
